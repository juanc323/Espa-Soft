Tacticas de arquitectura

La arquitectura software se refiere a una estructuración de un sistema que se crea en etapas tempranas de desarrollo, está representa un diseño de nivel alto del sistema que tiene dos propósitos primarios, satisfacer y servir como guía de desarrollo.

“La calidad no es un acto, es un habito ” Aristóteles

Por que usar las tacticas de arquitectura

La arquitectura de software es importante ya que la manera en que se estructura un determia si sistema tiene impacto directo en la capacidad de este para satisfacer lo conocido como atributos de calidad del sistema, como lo son el desempeño, la usabilidad,modificabilidad, estos son sus atributos.

Usabilidad

Se relaciona con la facilidad con la que el usuario de un sistema especifico realiza y ejecuta una tarea deseada y el tipo de asistencia que se le brinda al mismo, la usabilidad comprende áreas como lo son las funciones del sistema de aprendizaje, el uso eficiente de un sistema, el minimizar del impacto de errores en el sistema, adaptación a las necesidades el usuario y por ultimo incrementar la confianza del usuario en el sistema al igual que la satisfacción de este.

Testeabilidad

La capacidad de prueba de un software refiere a la facilidad con la que se puede realizar la muestra de las fallas de este, refiriéndose a la probabilidad. Asumiendo que el software tiene por lo menos una falla, pronosticando que fallara en su próxima ejecución de prueba. Por intuición un sistema es comprobable si sus fallas se detectan fácilmente, esto debe ocurrir lo más rápido posible para así mismo darle solución.

Seguridad

Es una capacidad del sistema, es como se protege los datos y la información de accesos no autorizados y a su vez proporciona acceso a personal autorizado. Acciones tomadas en contra del sistema informático con el fin de hacer daño es llamado ataque y puede tomar varias formas, puede ser un intento sin autorización de acceder a los datos o también servicios en búsqueda de modificar datos base o negar usuarios legítimos.

Rendimiento

Consiste en el tiempo y capacidad con el que cuenta un sistema de software para cumplir con los requisitos de tiempo, cuando ocurren eventos como interrupciones, mensajes o solicitudes de usuarios entre otros, el sistema debe generar una respuesta oportuna, la esencia de rendimiento es la caracterización de eventos que posiblemente pueden ocurrir y su respuesta es basada en el tiempo del sistema a esos eventos.

Modificabilidad

Tiene relación con el cambio, centrando el interés del costo y el riesgo al realizar cambios, para analizar esta opción el arquitecto debe tener en cuenta preguntas como: ¿ que puede cambiar?, en realidad se puede producir un cambio en cualquier aspecto del sistema, ¿ cuál es la probabilidad de cambio? El planificar un sistema para todos los cambios potenciales esto equivale a un gran costo, ¿ cuándo se realiza el cambio y con quién? Anteriormente se realizo un cambio al código fuente esto quiere decir que un desarrollador realizo un cambio que se aprobó y luego se implementó generando una nueva versión.

Interoperabilidad

Es el grado en que dos o mas sistemas pueden realizar intercambios de información útil utilizando interfaces de un contexto en particular, también cuanta con la capacidad de interpretación correcta de los datos que se están intercambiando. La interoperabilidad se ve afectada por los sistemas que desarrollan la interoperación.

Disponibilidad

Esta se refiere a una propiedad del software lista para llevar a cabo una determinada tarea cuando esta se requiera, es una perspectiva amplia que abarca normalmente la confiabilidad. Es basada en el concepto de confiabilidad al agregarle una noción de recuperación, refiriéndose a cuando el sistema se rompe, regenerándose a sí mismo, esto ejecutándose por varios medios. La disponibilidad está estrechamente relacionada con la seguridad, cuando en un sistema se presenta una falla por culpa de un ataque de un ataque de negación.
